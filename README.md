# CSS Module Type Definitions

[![npm version](https://img.shields.io/npm/v/css-module-type-definitions.svg)](https://www.npmjs.com/package/css-module-type-definitions)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

Automatically generate TypeScript type definitions for CSS Modules with full type safety and IDE autocompletion.

## Features

- 🎯 **Type Safety**: Get TypeScript types for all your CSS module class names
- 🔄 **Automatic Generation**: Generates `.d.ts` files automatically during development
- ⚡ **Vite Plugin**: Seamless integration with Vite's dev server and HMR
- 🎨 **Naming Conventions**: Multiple naming convention strategies (camelCase, dashes, custom)
- 🌳 **Recursive Search**: Scans entire project directory trees
- 🔧 **Customizable**: Flexible configuration options
- 📦 **Zero Config**: Works out of the box with sensible defaults

## Installation

```bash
npm install css-module-type-definitions
# or
yarn add css-module-type-definitions
# or
pnpm add css-module-type-definitions
```

## Usage

### Vite Plugin

The recommended way to use this package is as a Vite plugin:

```typescript
// vite.config.ts
import { defineConfig } from 'vite';
import { pluginCssModuleTypeDefinitions } from 'css-module-type-definitions';

export default defineConfig({
  css: {
    modules: {
      localsConvention: 'camelCase',
    },
  },
  plugins: [
    pluginCssModuleTypeDefinitions({
      extension: 'module.css',
      localsConvention: 'camelCase',
    }),
  ],
});
```

### Usage in Components

Once configured, import your CSS modules as usual and enjoy full type safety:

```typescript
// Button.tsx
import styles from './Button.module.css';

// TypeScript knows about all your class names!
export const Button = () => (
  <button className={styles.button}>
    {/* Autocomplete works! */}
    <span className={styles.icon}>✓</span>
  </button>
);
```

```css
/* Button.module.css */
.button {
  padding: 10px 20px;
  background: blue;
}

.icon {
  margin-right: 5px;
}
```

This automatically generates:

```typescript
/* Button.module.css.d.ts */
/* eslint-disable @typescript-eslint/naming-convention */
/*
 *  This file is automatically generated by css-module-type-definitions
 */

export type Classes = 'button' | 'icon';
export type Locals = { readonly [c in Classes]: string };

export declare const button: string;
export declare const icon: string;

declare const __default: Locals;
export default __default;
```

## Configuration

### Options

| Option             | Type                 | Default         | Description                                                                      |
| ------------------ | -------------------- | --------------- | -------------------------------------------------------------------------------- |
| `rootDirectory`    | `string`             | `process.cwd()` | Root directory to search for CSS module files                                    |
| `extension`        | `string`             | `'module.css'`  | File extension pattern to match (e.g., `'module.css'`, `'css'`, `'module.scss'`) |
| `localsConvention` | `string \| function` | `'camelCase'`   | Naming convention for exported class names                                       |

### Naming Conventions

#### `'camelCase'` (default)

Exports both the original kebab-case name and its camelCase version:

```css
.my-button {
}
```

```typescript
// Both available:
styles['my-button']; // ✓
styles.myButton; // ✓
```

#### `'camelCaseOnly'`

Exports only the camelCase version:

```css
.my-button {
}
```

```typescript
// Only camelCase:
styles['my-button']; // ✗
styles.myButton; // ✓
```

#### `'dashes'`

Converts only dashes to camelCase (preserves underscores):

```css
.my-button {
}
.my_class {
}
```

```typescript
styles['my-button']; // ✓
styles.myButton; // ✓
styles.my_class; // ✓ (underscore preserved)
```

#### `'dashesOnly'`

Like `'dashes'`, but exports only the converted version:

```css
.my-button {
}
```

```typescript
styles['my-button']; // ✗
styles.myButton; // ✓
```

#### Custom Function

Provide your own transformation function:

```typescript
pluginCssModuleTypeDefinitions({
  localsConvention: (original, generated, filePath) => {
    // Use SCREAMING_SNAKE_CASE
    return original.toUpperCase().replace(/-/g, '_');
  },
});
```

## Advanced Usage

### Multiple Extensions

Configure for different CSS preprocessors:

```typescript
// For SCSS modules
pluginCssModuleTypeDefinitions({
  extension: 'module.scss',
});

// For Less modules
pluginCssModuleTypeDefinitions({
  extension: 'module.less',
});
```

### Specific Directory

Scan only a specific directory:

```typescript
pluginCssModuleTypeDefinitions({
  rootDirectory: './src/components',
  extension: 'module.css',
});
```

### Custom Naming

Use a custom naming convention:

```typescript
pluginCssModuleTypeDefinitions({
  localsConvention: (original, camelCase, file) => {
    // Add prefix based on file location
    if (file.includes('/components/')) {
      return `c_${camelCase}`;
    }
    return camelCase;
  },
});
```

## How It Works

1. **Scanning**: The plugin scans your project for CSS module files matching the specified extension
2. **Parsing**: CSS files are processed with PostCSS and CSS Modules plugins
3. **Extraction**: Class names are extracted from the `:export` pseudo-selector
4. **Generation**: TypeScript declaration files (`.d.ts`) are generated with exported types
5. **HMR**: During development, files are regenerated automatically when CSS modules change

## Requirements

- Node.js >= 24.0.0
- Vite (for plugin usage)
- TypeScript (recommended)

## TypeScript Configuration

Ensure your `tsconfig.json` is configured to recognize CSS modules:

```json
{
  "compilerOptions": {
    "types": ["vite/client"]
  }
}
```

## Examples

### BEM Notation

```css
.block {
}
.block__element {
}
.block__element--modifier {
}
```

```typescript
// With 'camelCase'
styles.block; // ✓
styles.block__element; // ✓
styles.block__elementModifier; // ✓
```

### Pseudo-classes

```css
.button:hover {
}
.button:focus {
}
```

```typescript
styles.button; // ✓ (pseudo-classes are ignored)
```

### Global Selectors

```css
:global(.global-class) {
}
.local-class {
}
```

```typescript
// Only local classes are exported
styles.localClass; // ✓
styles.globalClass; // ✗
```

## Troubleshooting

### Types Not Updating

If types aren't updating after changing CSS:

1. Restart the Vite dev server
2. Check that the file extension matches your configuration
3. Verify the file is within the `rootDirectory`

### Import Errors

If TypeScript can't find the module:

```typescript
// Add to vite-env.d.ts or global.d.ts
declare module '*.module.css' {
  const classes: { readonly [key: string]: string };
  export default classes;
}
```

### No Autocomplete

Ensure your IDE's TypeScript language service is running and restart it if needed.
